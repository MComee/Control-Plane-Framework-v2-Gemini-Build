# Project Constraints

This file lists the critical constraints that govern the development and operation of the project. These constraints may be technical, regulatory, ethical, or resource-based.

**Guidance for Content:**
*   **Technical Constraints:** e.g., specific technology stacks, performance requirements, compatibility needs.
*   **Regulatory/Compliance:** e.g., data privacy laws (GDPR, CCPA), industry standards.
*   **Resource Limitations:** e.g., budget, team size, timeframes.
*   **Ethical Considerations:** e.g., AI safety, bias mitigation.
*   **Architectural Constraints:** e.g., must be microservices-based, must integrate with existing systems.

Adherence to these constraints is non-negotiable and must be considered in all planning and execution phases.
