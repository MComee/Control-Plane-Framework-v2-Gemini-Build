# Definition of Done

This document defines the criteria that must be met for any piece of work (feature, task group, task) to be considered complete and "done."

**Purpose:**
To establish a clear, shared understanding of quality standards and completeness, ensuring consistency and preventing incomplete work from being merged or deployed.

**Criteria for "Done":**

*   **Task Completion Criteria Met:** All specific completion criteria defined in the task definition file (`project/docs/tasks/task-id.md`) have been satisfied.
*   **Validation Successful:** All validation expectations outlined in the task definition have been successfully met.
*   **Evidence Provided:** All required evidence of work completion (logs, artifacts, test results) has been generated and stored in the appropriate locations (e.g., `project/evidence/`).
*   **No Unresolved Blocks:** All dependencies are met, and no blocking issues remain that would prevent the work from being fully functional or integrated.
*   **Adherence to Scope:** The work performed strictly adhered to the allowed scope and did not introduce unauthorized changes.
*   **Code Quality (if applicable):** If code was modified, it meets defined coding standards, is well-commented where necessary, and passes all relevant linters/formatters.
*   **Documentation Updated:** Any necessary updates to project documentation (e.g., feature descriptions, task definitions) have been made.
*   **Approved by Stakeholder (if applicable):** For significant deliverables, formal approval from relevant stakeholders may be required.

This definition serves as the ultimate benchmark for task and feature completion.
