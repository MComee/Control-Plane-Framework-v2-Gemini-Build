# Project Features

This directory contains detailed definitions for each feature of the project. Each feature should be defined in its own Markdown file within this directory.

**Purpose:**
Features represent significant capabilities or user-facing functionalities of the project. They are derived from the project's core vision and are broken down into task groups and atomic tasks for implementation.

**Guidelines for Feature Files:**
*   Each file should follow the `framework/templates/feature-template.md`.
*   Feature definitions are considered "Protected" artifacts and should not be modified by execution tools.
*   New features should be added through the planning process, ensuring alignment with the project vision and constraints.
*   Feature files help organize work and provide context for task groups and tasks.
