# Overview

The Control Plane Framework v2 is designed to manage AI-assisted software development by establishing a clear, repository-native control layer. It operates on the principle of "repository truth," ensuring that the codebase and its associated documentation serve as the definitive source of intent and progress.

Key principles of this framework:

*   **Repository Control, Not AI Control:** This framework does not aim to control the internal decision-making processes of AI models. Instead, it directs their interaction with the project by defining explicit rules, scopes, and data sources within the repository.
*   **Preservation of Intent:** By centralizing project vision, features, tasks, and priorities in version-controlled files, the framework ensures that development efforts remain aligned with the original intent.
*   **Execution Alignment:** It provides precise handoffs and boundaries for execution tasks, minimizing scope creep and ensuring that work is performed as intended.
*   **Single Project Focus:** Each instance of this framework is dedicated to controlling exactly one project, ensuring clarity and preventing cross-project interference.
